DESCRIPTION

r.icesheet reconstructs the equilibrium surface of a perfectly-plastic ice sheet from a basal shear-stress field and a mapped ice margin, using pyICESHEET (a Python port of Evan J. Gowan's ICESHEET). There is no time evolution: given the margin, the basal shear stress, and the bed, the ice-surface elevation is fully determined by the stress balance tau_b = rho_i g H |grad S|. Flowlines are integrated inward from the margin, raising the surface according to the perfectly-plastic slope relation until each successive elevation contour is reached; where flowlines converge, ice divides form, and where contours pinch, domes and saddles separate.

NOTES

  • The shear_stress raster should be pre-smoothed (for example with r.resamp.bspline): the solver differentiates the field, so sharp polygon-edge steps must be smoothed first. This smoothing is intentionally a separate step, not part of the model. If the shear stress originates as polygons, rasterize it (v.to.rast) and then smooth it.
  • All inputs must share the same projected (metric) coordinate system, and the computational region defines the working grid: set it with g.region before running. Coarser regions and larger spacing run faster.
  • pyICESHEET and its dependencies (NumPy, SciPy, Shapely) must be installed in the Python interpreter used by GRASS.

EXAMPLE

# Rasterize and smooth a polygon shear-stress map
v.to.rast input=shear_stress output=tau_raw use=attr attribute_column=shear_stre
r.resamp.bspline input=tau_raw output=tau method=bicubic

g.region raster=bed -p
r.icesheet bed=bed shear_stress=tau margin=ice_margin \
    surface=ice_surface thickness=ice_thickness \
    spacing=5000 interval=100 max_elevation=4000
